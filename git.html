GIT

Concepto

Git es un software de control de versiones pensando en la eficiencia, la confiabilidad y compatibilidad del mantenimiento de versiones de aplicaciones cuando estas tienen un gran número de archivos de código fuente. Su propósito es llevar registro de los cambios en archivos de computadora y coordinar el trabajo que varias personas realizan sobre archivos compartidos en un repositorio de código.


GitHub

GitHub es una plataforma de desarrollo colaborativo para alojar proyectos utilizando el sistema de control de versiones Git. Se utiliza principalmente para la creación de código fuente de programas de ordenador. El código de los proyectos alojados en GitHub se almacena generalmente de forma pública.


Utilidades de GitHub

Esta plataforma cuenta con algunas herramientas o funciones que hacen que nuestra experiencia aquí sea óptima y que hacen de la página un lugar totalmente colabroativo donde toda la comunidad tiene la posibilidad de interactuar con los demás creadores.

Tres de estas funcionalidades son:

  • Fork: Esta herramienta nos permite, mediante un simple botón, copiar un repositorio de cualquier creador en nuestra cuenta y tenerlo como si fuese nuestro, pudiendo modificar todo cuanto queramos de él, lo cual resulta muy útil,por ejemplo, si necesitamos hacer algún programa en concreto que un usuario ya ha realizado o para hacer uso de la siguiente herramienta que explicaremos.

  • Pull request: Gracias a esta opción podremos optimizar, corregir errores o añadir funcionalidades a cualquier programa que se encuentre en la página, para ello solo tendremos que realizar el fork antes visto, clonar el repositorio al nuestro local y modificar aquello que deseemos para despues volver a subir a GitHub el archivo que hemos modificado y enviarle un pull request al autor solicitando que éste haga merge a ambas versiones para que nuestros cambios aparezcan en el repositorio.

  • Organizaciones: Esta opción nos da la posibilidad de crear grupos de trabajo (como el que hemos formado para este trabajo) para que todos los integrantes puedan participar en el proyecto teniendo un lugar común en el que se guardarán todos los archivos de forma privada pudiendo acceder solamente los que formen parte de dicha organización.


Comandos principales

  • git init [nombre del proyecto]: Crea un nuevo repositorio local con el nombre especificado

  • git clone [url]: Descarga un proyecto y toda su historia de version

  • git status: Enumera todos los archivos nuevos o modificados que se deben confirmar

  • git diff: Muestra las diferencias de archivos que no se han enviado aún al área de espera

  • git add [archivo]: Toma una instantánea del archivo para preparar la versión

  • git commit -m "[mensaje descriptivo]: Registra las instantáneas del archivo permanentemente en el historial de versión

  • git push [url]: Sube al repositorio remoto los cambios efectuados en la rama en la que nos encontremos
